Multi-agent system · A2A protocol · human-in-the-loop

A team of specialized agents that turns a regulation into a compliance gap report.

RegLens is a multi-agent system. Five purpose-built agents — across two frameworks, talking over the open A2A protocol — read a regulatory PDF and your control matrix, extract every obligation, check it against your policies, score the gaps by risk, and draft an auditable report. A human reviewer signs off before anything is finalized.

The problem

Compliance review is slow, manual, and stale by definition

Every new circular burns weeks of senior compliance time — and the answer is out of date the moment a policy changes.

⏳

Weeks per circular

A regulator publishes an update. Someone reads every clause, maps it to internal controls by hand, and writes it all up. Repeat for the next circular.

🔍

Easy to miss a clause

Obligations are buried in dense legal prose. Manual review misses edge cases — and a missed obligation is a real regulatory exposure.

📉

Stale the moment it ships

Policies drift. The report you finished last month no longer reflects reality, and there's no cheap way to re-run it.

How it works

Six steps, fully traceable end to end

Each step is a distinct agent or node. Every LLM output is schema-validated — no free-text JSON parsing anywhere in the pipeline.

1

Ingest the regulation

A Gemini multimodal agent reads the PDF and extracts a structured list of obligations — each with its clause, section, page, and type.

2

Retrieve matching policies

For each obligation, a pgvector semantic search pulls the most relevant controls from your matrix as candidate matches.

3

Analyze the gap

Claude classifies each obligation against its matches — COMPLIANT, PARTIAL GAP, GAP, or NOT APPLICABLE — with reasoning.

4

Score the risk

A Gemini agent applies a domain risk rubric to rate each gap CRITICAL / HIGH / MEDIUM / LOW, with a justification.

5

Draft the report

Findings are aggregated into a structured compliance report — JSON and markdown — and the pipeline pauses for review.

6

Human approves or edits

A compliance reviewer approves, rejects, or overrides any finding. Every edit is captured in the audit log before the report is final.

The interesting part

Agents talk over A2A — and the seam is observable

LangGraph and Google ADK never share a process. They communicate over A2A (Agent-to-Agent) — a JSON-RPC 2.0 protocol — which makes the cross-framework boundary explicit, typed, and traceable.

  • ①
    Self-describing Agent Cards Every agent publishes its skills and capabilities at /.well-known/agent-card.json — discovered and cached at startup.
  • ②
    JSON-RPC 2.0 over HTTP A minimal, honest transport — message/send and tasks/get — instead of an opaque SDK abstraction.
  • ③
    An OTel span on every call Request bytes, response bytes, status, retry count, and model are recorded at the boundary. The cross-framework hop is never a black box.
  • ④
    Resilient by construction Tenacity-backed retries with backoff, plus idempotency keys (run_id + node + obligation_id) so a retry never duplicates work.

Five services around one database. The supervisor orchestrates; the two ADK agents sit behind the A2A seam; Postgres makes every run durable and resumable.

Run it yourself

Everything runs locally with Docker Compose. Bring a Gemini and an Anthropic API key.

# 1 — configure and start the stack
cp .env.example .env
docker compose up -d

# 2 — submit a regulation
curl -X POST localhost:8000/runs \
  -F pdf=@circular.pdf -F domain=banking
